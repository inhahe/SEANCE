MIDI Input & Routing

Home Graph Basics Signals

SEANCE treats every input device — the computer keyboard, hardware MIDI keyboards, drum pads, network MIDI clients, virtual ports — as a node in the graph. You wire the input node to wherever you want the events to go. There's no hidden routing, no "arm" flag to hunt for; the cable is the routing.

The Computer Keyboard node

On a fresh project SEANCE automatically creates a Computer Keyboard node at the top-left of the graph. This represents the keys on your computer keyboard being used as a MIDI controller. It has a single MIDI output pin, which is pre-wired to the MIDI Track that's also placed by default.

Key mapping:

  • A-L — white keys
  • W-P — black keys
  • Z / X — shift octave down / up
  • Shift + key — play louder (velocity 120)
  • Alt + key — play softer (velocity 50)
  • Plain key — medium velocity (90)

Click the Keyboard MIDI toggle in the main toolbar to enable/disable computer-keyboard input. Disabling is useful when you want to use A-Z for other shortcuts without triggering notes.

Hardware MIDI devices

On a new project, SEANCE shows a wizard listing every MIDI input device currently detected on your system. Check the ones you want to use and click Add Selected — each checked device becomes its own MIDI Input node in the graph, which you can then wire to a track or synth.

You can re-open this wizard at any time via Options → Add MIDI Input Device…, in case you skipped one, want to add a newly-connected device, or deleted an input node and want it back.

Hotplug

If you plug in a new MIDI device while SEANCE is running, a small dialog will pop up asking if you want to add it to the graph. Click Add to drop a new MIDI Input node for the device at a default position, or Ignore to dismiss. Ignored devices stay ignored until the next restart or until you run the wizard again.

MIDI Tracks receive live input

Every MIDI Timeline node has a MIDI In pin on its left side. When you wire an input node's MIDI Out to a MIDI Track's MIDI In, the track receives live events alongside its clip playback. Both get merged into the track's MIDI Out, which is then sent to whatever's downstream (usually a synth).

You can skip the MIDI Track entirely and wire the input node directly to a synth if you just want to audition without recording. The Track layer is useful when you want to capture your playing to a clip — see the piano roll docs for editing captured notes.

CC Learn (MIDI mappings)

Any CC event from your controller can be mapped to any parameter on any node via MIDI Learn. Right-click a knob/slider → MIDI Learn → move your controller's knob → mapping captured. From then on, that physical knob moves that parameter.

Learned CC events are filtered out of the cable routing — if you've mapped CC#1 to a filter cutoff on a specific synth, moving the mod wheel only moves that cutoff. Notes, pitch bend, aftertouch, and unmapped CCs still pass through the cables normally.

Pitch bend, mod wheel, sustain pedal

The built-in synths now handle standard controller messages:

  • Pitch bend — ±2 semitones, applies to all sustained notes in real time
  • Mod wheel (CC#1) — drives a default 6 Hz vibrato whose depth scales with the wheel position. Can be disabled per-synth by setting the Vibrato param to 0, freeing CC#1 to be MIDI-Learned elsewhere.
  • Sustain pedal (CC#64) — holds notes through their release stage until the pedal comes up. Works on Piano, Waveform, Sampler, Drum Machine, and SoundFont synths.
  • Velocity — scales per-note volume. Each synth has a Vel Sens param (0..1) that controls how much velocity affects loudness. Set to 0 to ignore velocity entirely.

What if my keyboard doesn't play?

Check that:

  1. The Computer Keyboard (or device) node is present in the graph
  2. A cable runs from its MIDI Out to a MIDI Track or synth
  3. That target is wired (directly or through a chain) to the Master Out
  4. For computer-keyboard input specifically: the Keyboard MIDI toggle in the main toolbar is enabled, and the main window has keyboard focus (click the graph area)
